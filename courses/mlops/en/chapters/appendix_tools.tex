\section{Quick Decision Guide}
\label{app:decision-guide}

\begin{bestpractice}[title=\textbf{Recommended starter stack}]
For a team beginning its MLOps journey, a practical starting stack is:
\begin{description}
  \item[Environment] Conda (GPU) or uv (CPU-only)
  \item[Version control] Git + DVC
  \item[Experiment tracking] MLflow (self-hosted) or W\&B (cloud)
  \item[Containerization] Docker + Docker Compose
  \item[Serving] FastAPI (custom) or BentoML (standardized)
  \item[CI/CD] GitHub Actions + CML
  \item[Monitoring] Evidently AI + Prometheus + Grafana
  \item[Orchestration] Add Prefect or Dagster when pipelines grow
\end{description}
Start simple and add complexity only when the team outgrows simpler
tools. Premature adoption of Kubernetes or Kubeflow creates more
overhead than value for small teams.
\end{bestpractice}
